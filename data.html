Jak se vlastně ukládají v programu data? V proměnných. U každé proměnné musíme kromě názvu deklarovat i její typ. Od typu proměnné vyplývá i jaké data budou uloženy v proměnné Deklarace proměnné vypadá takto: datovytyp nazevpromenne; a definice napříkalad takto nazevpromenne = 5; ale i například nazevpromenne = jinapromenna; Zde je tabulka datových typů:
Název Typu: Počet bajtů: Rozsah:
short int 2 -32,768 do 32,767
int 4 -2,147,483,648 do 2,147,483,647
long int 8 -2,147,483,648 do 2,147,483,647
long long int 8 -(2^63) do (2^63)-1
char 1 -127 do 128, nebo znak
float 4 -
double 8 -
long double 12 -
Všechny (až na char, kam jde uložit i znak) jsou číselné typy, kam můžete ukládat čísla v rozsahu uvedeném v tabulce. Důležité je že jediné typy kam se dají ukládat desetinná čísla jsou: float, long double a double. Proto u nich není rozsah. Pokud před datový typ napíšete unsigned rozsah se zdvojnásobí +1, ale nepůjdou do něj ukládat záporná čísla. Nyní upravíme náš predešlý kód pomocí deklarací a definicí proměnných.

    #include < iostream >
    #include "naseknihovna.h"
    using namespace std;

    int main(void)
    {
      int a;
      int b;

      a = 5;
      b = 4;
      cout << scitej(a,b) << endl;       return 0;
    }

Výsledek bude stejný, pouze máme výhodu že pokud chceme změnit jedno z čísel stačí a = 5 upravit například na a = 6 a výsledek bude místo 10 Navíc proměnné můžeme používat v kódu vícekrát. Pokud nechceme proměnnou měnit a chceme aby nešla měnit kvůli vyvarování se chybám napíšeme před ní v deklaraci const
Pro zajímavost. Nejdelší možná deklarace proměnné v C++ :

  alignas (alignof(decltype(typeid(const volatile unsigned long long int)))) static thread_local inline constexpr volatile const decltype(typeid(const volatile unsigned long long int).name())* const a{};
